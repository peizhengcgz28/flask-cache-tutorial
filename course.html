Part 3 装饰器 — Python 的 @ 语法 — 10 分钟

💡 目标：理解 @xxx 是什么、怎么用。本项目大量使用了装饰器：@app.route()、@cache_response、@limiter.limit()、@measure_time。

3.1  装饰器本质："函数套函数"

装饰器就是一个函数，它接收一个函数作为参数，返回一个新函数。用 @ 语法糖简化写法。

# 装饰器原理：
# 普通写法：
def say_hello():
    print("你好！")

# 想加日志功能，但不改原函数
def log_wrapper(func):
    def wrapper():
        print("[日志] 调用前")
        func()                    # 调用原函数
        print("[日志] 调用后")
    return wrapper

say_hello = log_wrapper(say_hello)    # 手动包装
say_hello()

# 输出：
# [日志] 调用前
# 你好！
# [日志] 调用后
▶ 运行演示
点击查看装饰器原理

3.2  @ 语法糖

上面的 say_hello = log_wrapper(say_hello) 等价于：

@log_wrapper                    # 相当于 say_hello = log_wrapper(say_hello)
def say_hello():
    print("你好！")

say_hello()                      # 自动就有日志功能了

3.3  项目实战：@app.route()

Flask 最常用的装饰器，把函数注册为路由：

users_bp = Blueprint('users', __name__)

@users_bp.route("/users")            # GET /users 时触发
def get_users():

# 把 get_users 函数注册到 Flask 路由表中
# 当浏览器访问 /users 时，Flask 自动调用 get_users()

3.4  项目实战：@cache_response

本项目核心装饰器——缓存 API 结果：

# utils/cache_tools.py（简化版）
def cache_response(timeout=60):
    def decorator(f):                 # 接收被装饰的函数
        def wrapper(*args, **kwargs):
            key = make_cache_key(f.__name__)  # 生成缓存键
            cached = cache.get(key)           # 查缓存
            if cached:                     # 命中 → 直接返回
                cached["from_cache"] = True
                return cached
            result = f(*args, **kwargs)       # 没命中 → 调原函数
            result["from_cache"] = False
            cache.set(key, result, timeout)    # 存入缓存
            return result
        return wrapper
    return decorator

# 使用：

@cache_response(timeout=30)           # 缓存 30 秒

    ...

3.5  项目实战：@measure_time 和 @limiter.limit()

# @measure_time —— 统计接口耗时
def measure_time(f):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = f(*args, **kwargs)           # 执行原函数
        elapsed = time.time() - start
        result["elapsed_ms"] = round(elapsed * 1000, 2)
        return result
    return wrapper

# @limiter.limit("5 per minute") —— 限流
# 来自 Flask-Limiter 库：
# 同一 IP 每分钟最多访问 5 次，超限返回 429

# 项目中实际用法（api/users.py）：
@users_bp.route("/slow")
@limiter.limit("3 per minute")       # 每分钟 3 次
@measure_time                          # 统计耗时
def slow_query():
    time.sleep(2)                      # 模拟慢查询
    ...
💡 总结：装饰器就是在不修改原函数代码的前提下，给函数附加新功能。项目中的 @cache_response 加缓存、@measure_time 加计时、@limiter.limit 加限流——都是"附加功能"。

Part 4 项目模块化 + 架构详解 — 10 分钟

💡 目标：理解 6 层架构怎么拆、为什么拆、数据怎么流。

4.1  为什么拆成多文件？

📄好维护：改路由不动数据库
🔄好复用：到处 import
🔧好协作：每人负责一层
🔎好调试：问题定位更快

4.2  本项目目录结构

flask-cache-tutorial/ ├─ app.py # 入口：组装所有层 ├─ config/ │ └─ __init__.py # 配置：数据库、缓存、限流参数 ├─ core/ │ └─ __init__.py # 核心：Cache + Limiter 单例 ├─ database/ │ ├─ __init__.py │ ├─ connection.py # 数据库连接、建表、重置 │ └─ queries.py # SQL 查询：CRUD、日志、统计 ├─ service/ │ ├─ __init__.py │ └─ stats_service.py # 业务逻辑：组装统计数据 ├─ api/ │ ├─ __init__.py # 注册蓝图 │ ├─ system.py # 系统路由：/, /stats, /clear-cache │ └─ users.py # 用户路由：/users, /user/<id>, /search, /slow ├─ utils/ │ ├─ __init__.py │ ├─ response.py # api_response()、fix_json_response() │ └─ cache_tools.py # cache_response、measure_time 装饰器 └─ templates/ └─ index.html # 前端仪表盘

4.3  6 层架构职责

🌐api/路由层：接收 HTTP 请求、调用 service、返回 JSON 响应
↑ ↓
⚙service/业务层：复杂数据处理（如统计信息组装）
↑ ↓
🗃database/数据层：执行 SQL、操作 MySQL 数据库
↑ ↓
🛡core/核心：Cache + Limiter 单例，全局共享
↑ ↓
🔧utils/工具：通用函数（响应格式、缓存装饰器）
↑ ↓
📄config/配置：所有可调参数（数据库地址、缓存时间、限流频率）

4.4  数据流：访问 /stats 时各层做了什么

顺序层文件代码
1api/ api/system.py @system_bp.route("/stats")
def stats(): return build_stats_response()
2service/ service/stats_service.py def build_stats_response():
  data = db.get_statistics()
3database/ database/queries.py def get_statistics():
  cursor.execute("SELECT ...")
4service/ stats_service.py return {"total_users": ..., "total_requests": ...}
5api/ system.py return api_response(stats) → 浏览器收到 JSON
🌐 浏览器 GET /stats
↓
api/system.py → service/stats_service.py
↓
database/queries.py → MySQL
↑ 数据返回 ↑
api/system.py → JSON 响应（200 OK）
💡 核心原则——关注点分离（Separation of Concerns）：
每一层只做自己的事，不越界。
api/ 不直接写 SQL，database/ 不直接返回 HTTP 响应。
这样改数据库查询时，路由代码完全不用动。

4.5  app.py 如何组装一切

# app.py —— 模块化的大本营
from flask import Flask
from config import Config                # 1. 配置层
from core import cache, limiter          # 2. 核心层
from database.connection import Database  # 3. 数据库层
from database.queries import Queries      # 4. 查询层
from api import register_api_blueprints  # 5. API 层

def create_app():
    app = Flask(__name__)
    app.config.from_object(Config)          # 加载配置

    db = Database()                         # 连接 MySQL
    app.extensions['db'] = Queries(db)      # 注册查询对象

    cache.init_app(app)                     # 初始化缓存
    limiter.init_app(app)                   # 初始化限流

    register_api_blueprints(app)            # 注册所有路由
    return app

if __name__ == "__main__":
    app = create_app()
    app.run(debug=True)
